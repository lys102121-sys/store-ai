# AI CS Roadmap Status

이 파일은 작업 전 중복 구현을 막기 위한 현재 구현 현황표다. 새 기능을 시작하기 전에 관련 키워드로 코드 검색을 하고, 아래 상태와 맞지 않으면 이 파일도 함께 갱신한다.

## 이미 진행됨

- 카카오 로그인과 사용자별 데이터 분리
- 사용자당 store 1개 업데이트 구조
- 가게 정보 저장/불러오기
- 업종 선택/수기 입력, 업종별 입력 가이드, 정책 작성 도우미
- 상품 목록 학습
- 사장님 리뷰 말투 학습 `owner_reply_examples`
- CS 응대 말투 학습 `owner_cs_examples`
- 기존 tone 입력/프리셋 제거, 예시 기반 말투 우선순위 정리
- 문의 답변 생성
- 리뷰 답글 생성
- 리뷰 답글 일괄 생성
- AI CS 처리함
- 상태 탭: 확인 필요, 승인 대기, 답변 완료
- 플랫폼 필터: 전체, 수동 입력, 스마트스토어, 쿠팡, 배민, 요기요, 쿠팡이츠
- 더 보기/접기
- handling_type/risk_level/ai_reason 표시
- 데모 데이터 배지
- 안전 항목 일괄 승인
- missing info 학습과 관련 pending/needs_review 문의 재생성
- missing_info와 cs_message 중복 카드 숨김
- 가게 지식 저장, 수정, 삭제, 다시 사용, 관련 CS 문의 재검토
- 사장님 수정 답변 학습 후보 저장
- 반복 수정 패턴 기반 자동 완료 보류
- AI 업무 이력 로그
- 오늘의 AI CS 업무 요약
- 무료 체험 사용량과 유료 도입 상담 요청
- 관리자용 도입 상담 관리
- 쿠팡 연동 설정, 연결 테스트, 실제 문의 가져오기, 답변 등록 API 구조
- 스마트스토어 연동 설정, 연결 테스트, 실제 문의 가져오기, 답변 등록 API 구조
- 쿠팡/스마트스토어/배민/요기요/쿠팡이츠 샘플 데이터 흐름
- 플랫폼 상태 `local/synced/posted/failed`
- 가격/재고/출고/환불/예약/영업시간/건강/안전 추측 방지 가드
- 교차 업종 CS 안전 회귀 테스트
- 시작하기 화면 3분 체험 중심 단순화
- 흰색/회색/파란색/검정 중심 UI 팔레트 정리

## 일부 진행됨

- 실제 플랫폼 연동: 쿠팡과 스마트스토어는 문의 조회/답변 등록 구조가 있으나 실제 운영 검증과 예외 케이스 강화가 더 필요하다.
- 배민/요기요/쿠팡이츠 실제 연동: 샘플 리뷰와 플랫폼 필터는 있으나 실제 API 연동은 아직 없다.
- 결제/구독: 유료 도입 상담과 플랜 게이트는 있으나 실제 결제 연동은 아직 없다.
- 온보딩 UX: 시작하기는 단순화했지만 가게 설정과 AI CS 카드 정보 밀도는 더 줄일 수 있다.

## 다음 후보

1. 가게 설정 탭 복잡도 줄이기: 필수 입력 먼저, 상품/정책/말투/자동 처리는 접힌 고급 설정으로 정리.
2. AI CS 처리함 카드 정보 밀도 줄이기: 기본은 원문/AI 답변/주요 판단만, 상세 판단 이유와 플랫폼 정보는 접기.
3. 스마트스토어 실제 운영 예외 강화: 인증 실패, 중복 문의, 답변 등록 실패, 유료 플랜 안내를 더 명확히 한다.
4. 유료 결제 기반: 상담 요청 이후 실제 결제/구독 상태 전환 흐름을 설계한다.
